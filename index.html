Fokker Challenge: Smallest one-point basis in Lean 4

Searching for smallest one-point basis and verify in Lean4

View on GitHub Q&A

A one-point basis is a single lambda term that can generate the entire untyped lambda calculus by itself through repeated application.

The currently known smallest one-point basis for untyped lambda calculus (in terms of Fokker size) is α, discovered by John Tromp and Mtv Europe.

α = λλλ2 0 (1 (λ1))
B = λλλ2 (1 0) = α(α(α(α(α α) α))(α α) α)(α(α(α α)))
C = λλλ2 0 1 = α(α α α(α α α)(α α)(α α)) α α
K = λλ1 = α α(α(α α) α α α) α α
W = λλ1 0 0 = α α(α(α(α α) α))
I = λ0 = α(α(α(α α) α))(α(α α) α)
S = λλλ (2 0) (1 0) = α(α(α α(α α(α α))(α(α(α α(α α))))))α α

The Fokker size of a closed term is the number of abstractions and applications.

def fokker_size : (Term String) -> Nat
| bvar _ => 0
| fvar _ => 0
| abs t => 1 + fokker_size t
| app t1 t2 => 1 + fokker_size t1 + fokker_size t2

Our goal

  • Verify α is the smallest basis OR
  • Find the smallest basis

We use Lean 4 to formalize that all smaller closed lambda terms can't be a basis.

Our proof is based on cslib.

What we have done


There are 5420 terms with Fokker size < 7. We have proved 51% are not bases by 3 simple deciders.

  • Decider/EveryBvarUsed.lean: If every var is used, it couldn't be basis — unable to discard variable.
  • Decider/NoDuplicate.lean: If every var is used only 0 times or 1 time, it couldn't be basis — unable to duplicate variable.
  • Decider/All0.lean: In de Bruijn index, if every number is 0, it couldn't be basis — one variable is not enough, unable to construct complicated terms.

Next steps

  1. Deal with terms without redex first
  2. Verify "Two vars are not enough"
  3. Filter terms always terminate
  4. Filter terms always diverge
  5. Find more deciders: especially ability to reorder variables
  6. Investigate patterns of undecided terms

How AI Can Help (High Priority)

This project is quite suitable for AI-driven autonomous proof.

  1. Autonomous Solve Known Trival Cases: Use LLMs + Lean 4 tactics to prove individual terms are not bases.
  2. Automated Search Unknown Trival cases
  3. Pattern Recognition: Analyze undecided terms to identify common structures or properties, guiding the discovery of new deciders.
  4. Mass Analysis: Systematically analyze thousands of undecided terms in parallel.
  5. Our project is an excellent platform for testing LLMs' capabilities.
